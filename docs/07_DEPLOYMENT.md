## 2. Initial Deployment

### Step 1: Push to GitHub

```bash
# Initialize repo
git init
git add .
git commit -m "Initial commit: Hekaya Jewellery"
git remote add origin https://github.com/YOUR_USERNAME/hekaya.git
git push -u origin main
```

### Step 2: Connect Vercel

1. Go to [vercel.com](https://vercel.com) → Sign in with GitHub
2. Click **"Add New Project"**
3. Import your `hekaya` repository
4. Framework Preset: **Next.js** (auto-detected)
5. Click **Deploy**

### Step 3: Set Environment Variables

In Vercel Dashboard → Project → **Settings** → **Environment Variables**:

```
NEXT_PUBLIC_SUPABASE_URL          = https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY     = eyJhbGc...
SUPABASE_SERVICE_ROLE_KEY         = eyJhbGc...        ← ⚠️ NOT prefixed with NEXT_PUBLIC_
STRIPE_SECRET_KEY                 = sk_live_...       ← ⚠️ NOT prefixed with NEXT_PUBLIC_
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY= pk_live_...
STRIPE_WEBHOOK_SECRET             = whsec_...         ← ⚠️ NOT prefixed with NEXT_PUBLIC_
NEXT_PUBLIC_PAYPAL_CLIENT_ID      = your_client...
PAYPAL_CLIENT_SECRET              = your_secret...    ← ⚠️ NOT prefixed with NEXT_PUBLIC_

NEXT_PUBLIC_SITE_URL              = https://hekaya-Jewellery.com
NEXT_PUBLIC_SITE_NAME             = Hekaya Jewellery
NEXT_PUBLIC_WHATSAPP_NUMBER       = +971XXXXXXXXX
ADMIN_EMAIL                       = admin@hekaya-Jewellery.com
```

> Never prefix secret keys with `NEXT_PUBLIC_`. That exposes them to the browser. `SUPABASE_SERVICE_ROLE_KEY`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, and `PAYPAL_CLIENT_SECRET` must stay server-only.

### Step 4: Redeploy

After adding environment variables, redeploy:

- Go to **Deployments** → click the three dots on the latest → **Redeploy**

## 3. Custom Domain Setup

### Option A: Buy via Vercel (Simplest)

1. Vercel Dashboard → **Settings** → **Domains**
2. Search for `hekaya-Jewellery.com`
3. Purchase directly (~$12-15/year for `.com`)
4. Auto-configured, zero DNS setup needed

### Option B: External Domain (Namecheap, GoDaddy, etc.)

1. Buy domain from registrar of choice
2. In Vercel: **Settings** → **Domains** → Add `hekaya-Jewellery.com`
3. Vercel gives you DNS records to add:

```
Type    Name    Value
A       @       76.76.21.21
CNAME   www     cname.vercel-dns.com
```

4. Go to your domain registrar → DNS Settings → Add these records
5. Wait 5-30 minutes for propagation
6. Vercel auto-issues SSL certificate

## 4. Deployment Environments

### Branch Strategy

```
main        → Production   (hekaya-Jewellery.com)
staging     → Staging       (staging.hekaya-Jewellery.com)
feature/*   → Preview URLs  (auto-generated by Vercel)
```

### Environment Separation

| Environment     | Supabase                  | Payments                           | Domain                       |
| --------------- | ------------------------- | ---------------------------------- | ---------------------------- |
| **Development** | Local / free project      | Stripe `sk_test_` / PayPal sandbox | localhost:3000               |
| **Staging**     | Separate Supabase project | Stripe `sk_test_` / PayPal sandbox | staging.hekaya-Jewellery.com |
| **Production**  | Main Supabase project     | Stripe `sk_live_` / PayPal live    | hekaya-Jewellery.com         |

> [!IMPORTANT]
> Use a **separate** Supabase project for staging. Never test with production data.

## 5. Vercel Configuration

### `next.config.ts` — Image Optimization

```typescript
import type { NextConfig } from "next";
import createNextIntlPlugin from "next-intl/plugin";

const withNextIntl = createNextIntlPlugin();

const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "*.supabase.co",
        pathname: "/storage/v1/object/public/**",
      },
    ],
  },
};

export default withNextIntl(nextConfig);
```

> **Note:** In the current implementation, bilingual routing is **not** done with `next-intl` middleware. The app runs on **Next.js 15.5.15** with a single set of routes; the active locale lives in a Zustand store (`hekaya-locale`, default `ar`) + cookie, and `<html lang/dir>` is updated client-side via `useT()`. There is no `proxy.ts` / `middleware.ts` and no `[locale]` segment. Do NOT add an `i18n` block to `next.config`.

## 6. Performance Optimization

### Image Optimization

- Use Supabase Storage for product images and memory photos
- Set `sizes` prop for responsive loading

### Caching Strategy

| Content                   | Cache Duration | Strategy           |
| ------------------------- | -------------- | ------------------ |
| Static pages (About, FAQ) | 1 hour         | `revalidate: 3600` |
| Product pages             | 5 minutes      | `revalidate: 300`  |
| Product listing           | 2 minutes      | `revalidate: 120`  |
| Cart / Checkout           | No cache       | Dynamic            |
| Images                    | 1 year         | Immutable CDN      |

### Core Web Vitals Targets

| Metric                             | Target  | Priority |
| ---------------------------------- | ------- | -------- |
| **LCP** (Largest Contentful Paint) | < 2.5s  | High     |
| **FID** (First Input Delay)        | < 100ms | Medium   |
| **CLS** (Cumulative Layout Shift)  | < 0.1   | High     |

### Google Analytics Setup

```javascript
// src/app/layout.tsx — Add GA4 script
import Script from "next/script";

export default function RootLayout({ children }) {
  return (
    <html lang="ar" dir="rtl">
      <head>
        <Script
          src={`https://www.googletagmanager.com/gtag/js?id=${process.env.NEXT_PUBLIC_GA_ID}`}
          strategy="afterInteractive"
        />
        <Script id="google-analytics" strategy="afterInteractive">
          {`
            window.dataLayer = window.dataLayer || [];
            function gtag(){dataLayer.push(arguments);}
            gtag('js', new Date());
            gtag('config', '${process.env.NEXT_PUBLIC_GA_ID}');
          `}
        </Script>
      </head>
      <body>{children}</body>
    </html>
  );
}
```

## 9. Troubleshooting

| Issue                  | Cause                   | Fix                                                      |
| ---------------------- | ----------------------- | -------------------------------------------------------- |
| **Build fails**        | Missing env vars        | Add all vars in Vercel Settings                          |
| **API returns 500**    | Wrong Supabase keys     | Check `SUPABASE_SERVICE_ROLE_KEY`                        |
| **Images not loading** | Missing remote patterns | Update `next.config.js` `images.remotePatterns`          |
| **Payment failing**    | Test keys in production | Switch to live keys (Stripe `sk_live_` / PayPal Live ID) |
| **Slow load in UAE**   | No CDN region           | Vercel auto-handles, check image sizes                   |
| **SSL not working**    | DNS not propagated      | Wait 30 min, check DNS with `dig`                        |
| **404 on refresh**     | Missing route           | Check App Router folder structure                        |
